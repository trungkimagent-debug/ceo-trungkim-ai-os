# Module context notes

Create one concise note per tab/screen after reading that module's full relevant code.

Purpose: future edits should start from the module note, then inspect current source only where needed, saving time/tokens while keeping accuracy.

Required sections per note:

- Scope / tab label
- Source files and exact DOM section/module boundaries
- State variables and event handlers
- API calls / dynamic imports / required assets
- Safe edit points
- Do-not-touch boundaries
- Verification checklist
